關於 Kimai

從一項業餘愛好專案，到現在的世界領先的開放源始碼工時追蹤應用程式。讓我們來瞭解它的歷史。

2018

經過重寫的 Kimai 獲社群採用，資料移轉成功，使用率正在上升。

2019

建立了第一批外掛程式，以測試付費開放源始碼軟體對使用者的吸引力。

2020

Kimai 推出訂閱制的 SaaS 型式的服務，提供一套免費方案與兩套付費方案。

2023

辛勞的付出終將有所回報，Kimai 現在已成為我的 100% 全職工作。感謝您！

Kimai 是一款有著長遠歷史的工時追蹤軟體，第一個版本早在 2006 年就發佈了。為什麼我要提到這個呢？因為我已看過許多類似的專案時來時去——相信我，Kimai 會留下來。多年以來，開發 Kimai 是我的興趣；現在，Kimai 是我的事業，我從未像現在這麼有動力地不斷改進它。

我建構 Kimai 的目的只有一個：記錄、管理、評估、與核算工作時間。Kimai 支援無上限的使用者、客戶、專案、與工時紀錄項目，在一個安裝實例中就能夠輕易管理所有團隊。Kimai 已被翻譯為數十種語言，可以在任何具有網路存取與瀏覽器的裝置上執行，藉助了回應式設計，甚至在路途中時也可在您的行動裝置上執行。

您可以在瀏覽器中使用可自訂的收支單模版，根據工時單直接建立收支單。Kimai 還可以透過多種方式進行擴充，特別是由活躍的社群所建立的許多外掛程式。

無論您是自由工作者、小型組織、還是擁有數百名員工的公司，Kimai 都能夠幫助您管理業務。請依您的心意決定並利用體驗版本測試 Kimai。

誰在開發 Kimai？

Kimai 與 Kimai Cloud 由來自奧地利維也納的獨立軟體開發者 Kevin 開發與運營。

協作

客戶讚賞我們所提供的免費支援服務的便利，這種互動是我們業務不可或缺的核心。我很喜歡在 GitHub 上的社群交流、協作、與快節奏的開發工作。憑藉各位的寶貴反饋，我們長期不斷改進，使這段過程更加卓越非凡。

技術

身為開發者，我熱衷於測試新技術。如果這能改進您與 Kimai 的互動方式，我會認為這是雙贏。UI 與 UX 對我來說至關重要，因為軟體必須易於使用。資訊安全是基本要求，而開源軟體對許多企業來說是成功的關鍵因素。

工作的樂趣

Kimai 的開發是我喜愛的職業、無限可能的教育、與養家餬口的需要的理想結合。多虧了 Kimai，我有了更多時間與妻兒相處，這或許是您所能想到的最好的動力。

Kimai 的源起

Kimai 有著長遠的歷史，從 2006 年存在至今，以下將簡單介紹它的源起。

2006

Kimai 的第一個版本發佈，並逐漸獲得愈來愈多的使用者與開發者。

2009

最初的開發者 Torsten 想終止 Kimai 的開發。Kevin 接手並成為維護者。

2014

發佈 Kimai 1 的最後一個版本，開發停滯且消極。

Kimai 有著長遠的歷史，當 Torsten Höltge 的團隊在 2006 年開始開發時，它就已經遠遠領先於時代。當它開始採用回應式設計（藉助於大量的 JavaScript 魔法）的時候，這個詞甚至還沒被正式發明。 2009 年，當時 Torsten 因為時間不足而想終止開發，因此我接手了 Kimai 的開發，之後我在其他開源愛好者的協助下維護並擴充了 Kimai 數年。

在此同時，PHP 社群正取得長足的進步，而 Kimai 卻由於其歷史的原因而愈來愈落後。它的程式碼基礎存在著許多技術弱點，在多年的發展之後，已變得愈來愈難維護。 用現代化的框架重寫 Kimai 的想法已經存在了一段時間，同時受到了社群請求的推動，但一直沒有人有資源來推進這個想法。2017 年底，在多種因素結合之下，我抓住了時機開始開發革命性的下一步，「Kimai 2」。

最初的 Kimai 版本在 2006 年首次發佈，並已由世界各地的許多人所使用，我為此及其廣大的社群而自豪。感謝所有為 Kimai 作出頁獻的人。
