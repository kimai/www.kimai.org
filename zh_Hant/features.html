技術與業務支援

我確保對所有我所建構的應用程式提供支援，我將協助您與您的團隊。

一起合作

Kimai 能為您做什麼呢？

Kimai 不僅提供工時追蹤，還有其它重要的業務功能。

行動裝置友善

版面配置完全採用回應式設計，同時適用於桌上型電腦與行動裝置。每位使用者可以設定要顯示的資料縱行，可避免在小型裝置上需要橫向捲動等情況。

認證與安全

Kimai 支援外部身份識別提供者，可從多個提供者（例如 Google Workspace、Azure AD、Authentik）透過 LDAP 與 SAML 進行登入。可選擇使用 TOTP token 進行雙重要素身份認證。

外掛程式

Kimai 具有可擴充的設計，可透過外掛程式增加功能。想要建立外掛程式的開發人員能夠利用多個擴充點。

多使用者

Kimai 允許任意數量的使用者：每個已安裝實例都支援無上限的使用者與各種團隊角色（請參閱「權限」）。

多語言

Kimai 可以在地化為任何語言，目前已獲翻譯為至少 30 種語言。

團隊

可以建立團隊以將您的使用者合理分組，例如依部門分組。可將客戶與專案指派給這些團隊，以限制資料存取權。

客戶

管理您的客戶並設定地址、聯絡人、國家、貨幣、可見度（用於封存）、與更多其它欄位。

專案

透過訂單編號、預算、與可見度（封存用）來管理每位客戶的專案。

活動

活動用於將工作單元指派到工時單中的項目。可透過訂單編號、預算、與可見度（封存用）來管理每項專案的活動，也可建立全域活動來免去重複定義的繁瑣。

收支單

可自訂的收支單模版、項目分組選項、收支單編號、以及使用 PDF、DOCX 等多種檔案格式。

匯出資料

工時單資料能以 XLSX、PDF、CSV、或 HTML 檔案格式匯出共用。可以鎖定已匯出的紀錄以防止再被更改。

開銷管理

購買該外掛程式後，可以根據客戶、項目、與活動來追蹤開銷 ，以便分類並列入您的收支單。

SAML

利用 SAML 式單一登入，透過您現有的 SSO 身份識別提供者，例如 Google Workspace、Authentik、Microsoft Entra ID，來登入與註冊。

JSON API

具有能用以讀取資料的全面 JSON API，因此外部應用程式與自訂工具能與 Kimai 溝通。

雙重要素驗證（2FA）

Kimai 支援使用 TOTP token 進行雙重要素身份認證（2FA）：立即保護您的使用者帳號！

統計資訊

您的儀錶板會顯示有關已記錄工時與收集資料量等的統計資訊。而我們能持續新增更多種圖表的支援！

Docker

我們有用於開發的 Docker 映像，還有為正式用途所設定的多種 Docker-Compose 變體。

打卡鐘模式

不希望使用者任意登記工時表紀錄嗎？可改用打卡鐘模式，該模式只允許「打卡上班」與「打卡下班」的紀錄。

費率計算

費率可按工時或按次計算，可針對客戶、專案、活動、與各工時紀錄各別設定。

LDAP

將 Kimai 與您的公司目錄伺服器（LDAP 或 Active Directory）連接並使用您現有的身份驗證基礎架構。

版本遷移

支援匯入 Kimai v1 的資料，以便從您的工時追蹤工具無縫遷移。

進行中的紀錄數

可允許平行記錄任意次，也可限制為一次。

上午／下午格式

Kimai 使用在地化的 24 小時制格式。但若您較習慣上午／下午格式，您可調整相關設定。

僅列出持續時長

如果您當地的法律要求您只列出持續時長而不是列出開始與結束時間，您可以全域啟用本模式。

工時表篩選器

您可依使用者、客戶、專案、活動、與日期篩選所有的工時紀錄。

權限

可自訂的權限架構。指派不同的角色予您的使用者，例如一般使用者能記錄工時，組長能列印收支單與設定專案與活動，管理員能設定使用者與客戶。

使用者資料

個人統計資料與個別使用者的每小時費率、Kimai 顯示主題、密碼、與個人資料相片等設定。

使用者註冊

Kimai 支援使用者透過以電子郵件啟用帳號的流程來自行註冊。

密碼重設

忘記密碼了嗎？Kimai 可協助您透過電子郵件來重設密碼。

使用者通道

使用祕密網址與任何人分享客戶與專案的統計資料與工時單，可選擇以密碼保護。

灰暗模式

每個應用程式都需要灰暗模式，在傍晚或暗處時有助舒緩眼睛疲勞。

報告與分析

分析已記錄的工時，根據使用者、客戶、專案、活動、標籤、時間區間、等條件進行評級。

字典裡沒有限制

您用的 SaaS 軟體都有附帶限制？Kimai 不一樣，沒有這些限制。您只須為有活動的使用者負擔費用。我不對您設定限制，也不對沒有活動的使用者收費。您可建立無上限的使用者、客戶、專案、活動、工時單、等等。

隱私第一

我重視您與您的使用者的隱私與資料。不，這不是某個隱私政策的枯燥開場，而是我營運這個專案的方式。我不販賣資料。我遵守歐盟 GDPR 且會回答關於相關的第三方資料處理者的任何疑問。

Kimai 的資訊安全

Kimai 在應用程式的設計上實作了最佳實踐以避免常見的攻擊。自動化的資訊安全控制在任何軟體的開發流程中都不可或缺，能減少軟體建立到弱點偵測的所需時間。每次程式碼一有修改都會經過這些掃描。

平臺的資訊安全

軟體能有多安全，取決於執行軟體的系統有多安全。我用了多層防禦的方式設計伺服器與網路架構。雲端平臺上的使用者，資訊安全受多層防禦保護。
